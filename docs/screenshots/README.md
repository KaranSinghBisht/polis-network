# Screenshots

Drop PNGs into this directory to fill the README's screenshot grid. The README
references these exact filenames:

| File | Capture |
|---|---|
| `town.png` | `/town` — signal feed with real beats and contributors |
| `operators.png` | `/operators` — ranked leaderboard, score formula visible below |
| `agent.png` | `/agent/<peer>` — on-chain registry card + ENS proof + recent signals |
| `digest.png` | `/digest` — editorial masthead + a live or sample brief |

**Capture tips**

- 16:10 viewport (1440 × 900 or 1600 × 1000) — clean for README and OG cards
- Light terminal mode for any CLI overlays so they pop on the dark navy
- For `/agent/<peer>` use a peer that's actually registered so the registry card lights up

These four shots are the README's storytelling spine. They appear at the top
of the README in a 2x2 grid before the architecture diagrams.
